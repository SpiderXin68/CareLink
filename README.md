# 康邻 CareLink

面向社区的 AI 居家健康守护与慢病管理平台演示作品。项目聚焦“一老一慢一社区”，用可交互原型展示从居家监测、风险预警、社区医生随访、家属协同到上门护理的服务闭环。

## 运行方式

无需安装依赖，图标脚本已随项目放在 `vendor/` 目录。只看静态演示时可以直接打开：

`C:\Users\SpiderXin\Desktop\Codex\CareLink\index.html`

如果要接入真实华为健康数据，必须通过本地服务访问：

```powershell
node bridge-server.js
```

然后访问 `http://localhost:5173`。

手机端桥接 APK 需要把数据 POST 到电脑局域网地址，例如：

`http://192.168.0.194:5173/api/health/huawei`

## 作品内容

- 社区指挥台：展示试点社区健康 KPI、楼栋级风险热力图、今日异常事件队列、闭环流程。
- 医生工作台：展示居民健康档案、AI 风险分层、随访任务生成与处理。
- 居民端：老人模式界面，突出“今天是否安全”、服药提醒、复测提醒和一键求助。
- 家属端：展示授权下的远程看护、异常通知、关怀时间线和隐私权限。
- 护理端：展示上门服务任务、结构化采集、药盒/饮食/环境记录。
- 设备接入：展示华为手表/运动健康数据如何通过 Health Service Kit 接入心率、步数、睡眠和运动记录。
- AI 风险引擎：展示模型输入、风险输出和合规边界，明确 AI 不替代医生诊断。

## 演示脚本

1. 先打开“社区指挥台”，说明 CareLink 不是单点健康 App，而是社区级健康闭环平台。
2. 点击“模拟采集”，展示居家数据进入平台后，风险等级和任务队列会实时刷新。
3. 进入“医生工作台”，选择红色或橙色居民，讲清楚 AI 只负责分层、派单和建议，医生负责干预。
4. 进入“居民端”和“家属端”，展示大字模式、一键求助、家属协同和授权查看。
5. 进入“护理端”，说明线上预警如何转为线下上门服务。
6. 进入“设备接入”，点击“模拟授权接入”，展示华为运动健康数据进入 CareLink 统一健康档案。
7. 进入“AI 风险引擎”，强调异常检测、慢病风险分层、服务优先级调度和隐私合规。

## 华为手表接入说明

正式接入不是由网页直接读取手表，而是通过：

`华为手表 / 手环 → 华为运动健康 App → HMS Core Health Service Kit → 用户授权 → CareLink 接入服务 → 健康档案与风险引擎`

上线前需要完成：

- 在华为开发者联盟创建应用并集成 HMS Core Health Service Kit。
- 申请心率、步数、睡眠、运动记录等健康数据权限。
- 由居民本人完成授权，并支持撤回授权。
- 通过移动端或服务端接入层同步数据，静态网页原型只做交互演示。
- 将华为原始数据映射为 CareLink 统一字段，例如 `heartRateBpm`、`steps`、`sleepHours`、`workout`。

官方参考：

- [Huawei Health Service Kit](https://developer.huawei.com/consumer/en/hms/huaweihealth/)
- [Huawei Health App](https://consumer.huawei.com/en/mobileservices/health/)

## 参赛定位

一句话总结：

“康邻 CareLink”以社区为单位，面向老年人与慢病居民，接入居家健康监测设备和居民健康档案，通过 AI 风险分层、家庭医生随访、家属协同和数字孪生社区大屏，构建从预防、监测、干预、复诊到康复的智慧健康服务闭环，实现基层医疗减负、养老照护提效和居民健康普惠。

核心创新点：

- 从“单点设备监测”升级为“居民、家属、社区医生、养老护理、医疗机构”的服务闭环。
- AI 定位为基层医疗分诊助手，不替代医生诊断，降低合规风险。
- 社区健康数字孪生只展示楼栋和网格风险，兼顾治理效率与个人隐私。
- 老人端采用大字体、大按钮、少图表、一键求助，体现银发友好设计。

## MVP 落地范围

- 1 个社区，300 到 500 名居民。
- 重点纳入 100 名老人、100 名高血压或糖尿病患者、20 名独居老人。
- 接入 1 个社区卫生服务中心或家庭医生团队。
- MVP 功能限定为居民健康档案、血压/血糖/心率上传、AI 风险分层、用药和复测提醒、医生随访工作台、家属异常提醒。
